org.cojen.tupl.ext

Interface TransactionHandler


  • public interface TransactionHandler
    Handler for custom transactional operations. Undo operations are applied to roll back transactions, and redo operations are applied by recovery and replication.

    Brian S O'Neill

      • setCheckpointLock

        void setCheckpointLock(Database db,
                     Lock lock)
        Called once when the database is opened, providing a shared lock which is later acquired exclusively by checkpoints. Implementation is permitted (but not required) to use this lock for suspending transactional changes when checkpointStart is called. Simply hold the lock when making transactional changes, and then no additional locking is required when checkpointStart is called.

      • checkpointStart

        Object checkpointStart(Database db)
                       throws IOException
        Called to durably write all changes. Implementation must suspend all transactional changes (although transactions can be left open), and then initiate a concurrent checkpoint. The checkpoint must capture a snapshot of all changes at the point it was initiated, and it does not need to contain any changes afterwards. A recovery sequence rolls back to the last checkpoint, and then it applies all redo operations to catch up. Undo operations are applied only for transactions which did not commit.
        Returns:
        optional object to be passed to checkpointFinish
        Throws:
        IOException
